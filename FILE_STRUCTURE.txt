------------------------------------------------------------
IMPORTANT RULES (isse "right" structure maintain hoti hai)
------------------------------------------------------------
1. `app/` ke andar sirf ROUTES rakhein вЂ” koi bhi reusable component
   seedha kisi page folder ke andar mat banayein. Agar ek component
   sirf ek page use karta hai, to us page folder ke andar
   `components/` sub-folder banayein (jaise my-profile-extension/components
   mein already kiya gaya hai вЂ” yeh sahi pattern hai).

2. `components/` ke andar cheezein feature-wise group karein
   (hiring, hr-admin, leads, layout, ui, shared) вЂ” flat mat rakhein.

3. `lib/` sirf non-UI logic ke liye hai: API calls, formatters,
   constants, config. Koi bhi .tsx/JSX yahan mat rakhein.

4. Naya reusable hook banate waqt `src/hooks/` folder use karein
   (abhi missing hai, add karna recommended hai).

5. Shared TypeScript types/interfaces ke liye `src/types/` folder
   banayein, taaki types alag-alag files mein duplicate na hon.

6. Route groups вЂ” (auth), (dashboard), (super-admin) вЂ” ka use
   layout aur access-control alag rakhne ke liye already sahi hai,
   isi pattern ko follow karte rahein.

7. File naming: components PascalCase (Button.tsx), routes/pages
   lowercase-kebab (page.tsx andar folder ka naam kebab-case),
   utils/lib files camelCase (axios.ts, csv.ts).
